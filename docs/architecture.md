# Architecture

## Routing

Simple state-based routing in [src/App.jsx](../src/App.jsx) — no React Router.

| State value | Page rendered |
|---|---|
| `'home'` (default) | `DashboardPage` |
| `'solution-tables'` | `SolutionTablesPage` |

Navigate between pages via the `onNavigate` prop (calls `setPage`).

---

## Pages

**[src/pages/DashboardPage.jsx](../src/pages/DashboardPage.jsx)** — Light-themed full dashboard (BureauTrack branding). Default landing page.
- Left sidebar + main content layout, full viewport
- 5 KPI cards, 4 bureau acceptance rate cards, submission log table, quick actions, recent activity
- Fetches `cr045_submissionrequests` (top 100), `cr045_submissionapprovals` (top 20), `cr045_creditbureaus` (top 50) in parallel
- Sub-components defined inline: `KpiCard`, `BureauCard`, `StatusBadge`, `NavItem`

**[src/pages/SolutionTablesPage.jsx](../src/pages/SolutionTablesPage.jsx)** — Dark-themed raw data explorer.
- Tab bar for all 4 solution tables, dynamic column extraction, top 50 records per tab

## Data Layer

The auto-generated service file is at [src/generated/services/MicrosoftDataverseService.ts](../src/generated/services/MicrosoftDataverseService.ts) (1563 lines — do not edit directly).

**Key methods used:**
```ts
MicrosoftDataverseService.GetOrganizations()
// → IOperationResult<{ value: [{ Url, FriendlyName }] }>

MicrosoftDataverseService.ListRecordsWithOrganization(
  orgUrl, entitySetName,
  undefined, undefined, undefined, undefined,   // prefer, accept, metadata, sensitivity
  undefined, undefined, undefined, undefined,   // $select, $filter, $orderby, $expand
  undefined, $top                               // fetchXml, $top
)
// → IOperationResult<{ value: Record<string, unknown>[] }>
```

**IOperationResult shape:**
```ts
{ success: boolean; data?: T; error?: { message?: string } }
```

**Org URL resolution pattern** (used in both pages):
```js
MicrosoftDataverseService.GetOrganizations()
  .then(r => setResolvedOrgUrl(r.success && r.data?.value?.[0]?.Url || ORG_URL))
  .catch(() => setResolvedOrgUrl(ORG_URL))
// GetOrganizations() works in the Power Apps portal context;
// falls back to hardcoded ORG_URL for local pac code run
```

**OData metadata key filter** (used in both pages to strip non-data fields):
```js
const isMetaKey = k => k.startsWith('@') || k.startsWith('odata')
```
